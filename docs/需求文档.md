# 财报点评报告自动生成

## 痛点

我司可投股票占比同业偏高，管理要求方面需要研究员定期跟踪的事项多，例行跟踪分散研究精力。财务报告集中发布时期，跟踪点评工作量大，行业研究员很难全面覆盖。

## 目标

实现一个财报点评系统，针对不同行业的公司财报进行点评。

## 方案

- 沟通、确定分析重点：与行业研究员沟通，确认行业点评重点关注的财务指标以及指标的变动表示的业务含义
- 取数，精确提炼数据：从财务报表中收集关键指标数据，避免模型编造
- 提示词，精准表达业务：将内部研究员对指标的理解，融入提示词，实现知识的沉淀
- 格式，规范报告要求：按照研究员的历史报告格式和行文特点，生成报告

## 实现

### 研究员的理解

#### 核心指标和次要指标

- 核心指标往往是最直接影响股价涨跌的，换句话说一次财报上百个上千个经营指标，可能其中仅有1-2个指标是真正影响股价涨跌的
- 而次要指标主要起着辅助核心指标判断的因素，可以理解成主要是验证+辅助核心指标的作用，，来验证逻辑的作用

#### 行业特征

以下以计算机行业为例：

- 成长性（想象带来的）+高估值（甚至终局估值）+无法拆和预测报表
- 成长性：多数计算机公司整体上属于TMT，且是TMT高风偏（计算机一直是整个A股风偏高的代表），所以成长性，或者能想象的成长空间长期显著影响股票估值，而基本面特别是近些年偏弱
- 高估值：计算机公司普遍估值相比传统行业估值P/E倍数偏高，甚至出现了“终局估值法”，即假设公司在终局稳态能做多少收入/利润，然后按终局稳态给估值，行业最大标的金山办公即是这样，合合信息/达梦数据等标的也是。
- 无法拆和预测报表：计算机公司业绩可预测性很差，即便是美国的SaaS公司也是这样。多数计算机公司业绩调节+不同季度/年度间主观调节能力非常强。因此普遍适用于传统行业的细拆报表来做线性预测的方法在计算机不适用。

以上3点尤其是第3点，造就了计算机行业业绩点评特点是：卖方普遍甚至不搭模型，买方对业绩指标只看核心大略（同时买方对计算机都是交易心态，计算机行
  业长期机构持仓偏低）

#### 行业业绩共性核心指标和个性辅助指标拆解

- 共性核心指标：收入增速及细分板块收入增速。
  A）解释：一个典型的计算机公司可能包含多个业务，其中某些关键业务板块的增速可能是股价决定因素，其他板块即便增速不及预期也可以。
  B）注：首先这是由计算机行业成长性+高估值决定，在成长板块肯定核心要看增速。其次计算机公司成本基本是人力成本，人力成本核心就是人头x工资，这都是可以调节的，但增速没了调节也没用，甚至负向双杀，所以核心还是看增速。历史上主动调节成本的周期都不会带来股价上涨
   
- 共性核心指标：利润及其增速。
  A）解释：利润增速重要性弱于收入增速，因为反映成长性更弱/可调节更多/其他指标都可以辅助判断，但利润毫无疑问是次要指标
- 共性辅助指标：毛利率
  A）解释：主要用于辅助增速指标，在不同板块重要性不同时，要分板块分拆毛利率。如果出现毛利率x收入增速正向双击，是个非常好的情况。最基础的，毛利率体现了当前增速的潜力和健康程度，和公司竞争力和周期。）注：首先计算机公司主动调节收入的能力会很强，但毛利率相比收入增速可掩饰性低得多，且如果收入高增但毛利下滑，那么可能是以价换量。其次中国计算机公司多数收入都是项目制而不是产品制，因此项目制下基本上毛利率作假能力反而较低，且直接和行业周期相挂钩，还可以辅助判断行业周期来判断收入增速的确定性。
   
- 共性辅助指标：研发费用与研发费用率
  A）解释：研发费用特别是对质地较好的计算机公司是其增速/战略的先导性指标。主流的研发费用分：算力支出的固定成本+研发人员总工资。对传统的计算机公司前者甚至没有，研发人员在计算机公司中可能就是实际驻场客户的程序员。
  B）注：因此，很多计算机研究员在咨询计算机公司收入增速外都要咨询人头数量，两者甚至在很多行业信息化公司是完全相关的。而哪怕是AI公司，研发费用率的显著变化都是公司战略变化的先导指标。总结：很多时候研发费用不决定股价，但是对收入增速和公司战略有先导性影响，且造假程度很低。
   
- 共性辅助指标：销售费用率
  解释：项目制公司这代表商务费用，产品型公司反映投流费用。这也是反映增速的重要指标，一方面可能先于增速动而动，一方面若高增速但费用率下降，则代表增长良性。
- 个性辅助指标：合同负债
  解释：对于云计算等订阅制公司来说，合同负债反映了先确认订单但未确认收入的部分，则有很强的增长先导性指标。
   
- 个性辅助指标：存货季度环比变化绝对值
  对算力/服务器出售的公司来说，存货反映了先确认订单但未确认收入的硬件产品，是收入的先导性指标。
   
  总结：计算机公司常规而言最重要的增长性指标，增长指标核心看收入增速，任何计算机公司最重要的就是收入增速。其次辅助增长指标是毛利率/研发费用/销售费用，这三个重要性显著递减，实际上仅有产品订阅型公司当期研发/销售费用可能很影响短期业绩股价。但这三个都是对增长的辅助性指标，且可以对着推断利润增速。也因此虽然表观当然利润是核心指标，但是其实可调节性非常强，且显著由前面变量调节影响。

### 提示词

**指标优先级及分析要求**：

1. **收入增速及细分板块收入增速（核心）**

- 分析总收入增速及关键业务板块（如云计算、软件服务、硬件销售等）的增速。
- 说明核心增长驱动因素：若高增长板块占比提升，即使其他板块疲软也可视为积极信号；若增速放缓，需评估是否预示行业周期或竞争恶化。
- 结合行业背景（如政策支持、技术迭代）判断增速可持续性。

2. **利润及其增速（次要核心）**

- 分析净利润及调整后利润的增速，对比收入增速判断盈利质量（如利润增速是否低于收入增速，可能反映成本压力或补贴依赖）。
- 评估利润的可调节性（如人力成本控制、非经常性损益），避免被短期优化误导。

3. **毛利率（关键辅助）**

- 计算整体毛利率及分板块毛利率，结合收入增速分析：
- 若收入增速与毛利率同升（正向双击），表明竞争力增强或行业景气；
- 若收入增速高但毛利率下滑，警惕以价换量或竞争加剧；
- 若毛利率稳定但收入低迷，可能需求疲软或产品竞争力不足。
- 对项目制公司，毛利率需与行业周期挂钩（如政府项目毛利波动）。

4. **研发费用与研发费用率（战略先导指标）**

- 分析研发费用绝对值及占收入比重的变化：
- 研发费用大幅增长可能预示战略投入（如AI、新产品线），但需跟踪后续收入转化；
- 研发人员数量变化与收入增速的关联性（如人头增长但收入未跟进，可能效率下降）。
- 强调研发费用的低造假性，作为战略真实性的参考。

5. **销售费用率（增长健康度指标）**

- 对比销售费用率与收入增速：
- 销售费用率下降伴随高增速，表明增长良性（如品牌效应或复购提升）；
- 销售费用率激增但增速未跟上，可能市场拓展受阻或竞争激烈。
- 区分项目制（商务费用）与产品制（投流费用）公司的差异。

6. **合同负债（订阅制先导指标）**

- 针对云计算/SaaS公司，分析合同负债环比变化：
- 合同负债增长预示未来收入确定性高；
- 若合同负债停滞或下滑，需排查客户流失或订阅周期问题。

7. **存货季度环比变化（硬件公司先导指标）**

- 对服务器/算力硬件公司，分析存货绝对值变化：
- 存货增长可能反映订单储备充足，预示未来收入；
- 存货激增但收入未兑现，警惕滞销或供应链风险。

**报表结构要求**：

1. **核心结论**：概括公司成长性（收入增速为核心）、盈利趋势及关键风险。
2. **分项分析**：按指标优先级逐项展开，数据与行业对比（如同业公司、历史均值）。
3. **综合判断**：

- 增长质量：结合毛利率、费用率评估增速健康度；
- 战略有效性：通过研发/销售费用分析公司布局；
- 先导信号：合同负债/存货对未来收入的指示作用。

4. **投资建议**：
   明确看多/中性/看空逻辑，如“高收入增速+毛利提升”为强势买入信号，“增速放缓+费用率攀升”提示风险。

**示例语句参考**：

- “公司2023年总收入增速25%，其中云业务增速达40%驱动整体增长，但硬件板块增速仅5%拖累表现。”
- “毛利率同比提升2个百分点至65%，与收入增速形成正向双击，反映产品结构优化及定价能力增强。”
- “研发费用率升至15%，主要投入AI研发，虽短期压制利润，但可能为长期增长蓄力。”  **注意事项**：
- 优先基于最新季度/年度数据，避免过度依赖历史表现。  - 对异常数据（如收入增速与利润背离）需重点解释原因（如季节性、会计调整）。
- 强调计算机行业“成长性优于利润”的特性，避免孤立分析利润指标。
